# Phase 6A Topic Page Audit

Date: 2026-05-25

## Purpose

This audit identifies current route systems that behave like autogenerated topic pages instead of admin-controlled entity or guide records. The goal is to preserve indexed URLs while preparing these pages to render through the unified guide engine in later phases.

No redirects, sitemap changes, canonical changes, or frontend redesigns are included in this phase.

### B. Guide-Like Topic Pages To Convert

These pages are currently generated through `src/app/cities/[citySlug]/[seoPage]/page.tsx` and `src/lib/programmatic-seo.ts`. They should keep their public URLs, but the internal renderer should eventually use guide-engine content blocks.

| Current URL example | Source config slug | Classification | Future action |
| --- | --- | --- | --- |
| `/cities/muscat/best-places` | `best-places` | Guide-like topic | Convert to guide-engine rendering. |
| `/cities/muscat/things-to-do` | `things-to-do` | Guide-like topic | Convert to guide-engine rendering. |
| `/cities/muscat/travel-guide` | `travel-guide` | Guide-like topic | Convert to guide-engine rendering. |

### C. Thin, Duplicate, Or Low-Value Topic Pages

These pages are also generated through the same catch-all topic route. They are useful only when there is enough editorial or structured data behind them. Today they are keyword-matched content lists and are not directly admin-controlled.

| Current URL example | Source config slug | Risk | Future action |
| --- | --- | --- | --- |
| `/cities/muscat/best-cafes` | `best-cafes` | No cafe/restaurant entity model yet; keyword-only matching. | Merge into a guide or keep only with editorial guide support. |
| `/cities/muscat/best-restaurants` | `best-restaurants` | No restaurant admin model yet; can duplicate cafe/food content. | Merge, prune, or convert once restaurant entities exist. |
| `/cities/muscat/best-beaches` | `best-beaches` | Can duplicate destination/category pages. | Convert to guide-engine only when curated beach content exists. |
| `/cities/muscat/family-attractions` | `family-attractions` | Broad keyword matching can create thin pages. | Convert to guide-engine only with editorial family-travel blocks. |

There is no current route for `/cities/muscat/restaurants` or `/cities/muscat/beaches`; `family-attractions` exists only in the configured `/cities/[citySlug]/family-attractions` shape listed above.

## Current Topic Page System

- Route: `src/app/cities/[citySlug]/[seoPage]/page.tsx`
- Config: `src/lib/programmatic-seo.ts`
- Source data: city, destination, attraction, and guide records
- Rendering pattern: shared generated layout with destination, attraction, and guide grids
- Metadata: generated per config entry through `seoMetadata`
- Thin page handling: `robots: { index: false, follow: true }` when content is not meaningful

## New Preparation Utilities

- `src/lib/guide-engine-architecture.ts`
  - Classifies city topic slugs as guide-like or thin/duplicate.
  - Defines future guide-engine listing block concepts.
  - Defines block sources for destinations, cities, guides, restaurants, hotels, car rentals, attractions, and custom links.

- `src/lib/topic-page-audit.ts`
  - Exports an inventory of entity routes and city programmatic topic routes.
  - Marks whether routes are admin-controlled.
  - Marks whether indexed URLs should be preserved.
  - Lists conversion action and risk tags for each route family.

## SEO Safety Rules For Future Phases

- Preserve existing indexed topic URLs.
- Do not mass redirect topic URLs during conversion.
- Keep canonical paths unchanged until the replacement guide-engine renderer is live and verified.
- Prefer internal renderer swaps over route moves.
- Keep thin topic pages noindexed unless editorial depth improves.

## Future Migration Recommendations

1. Add an admin-controlled guide record for each high-value topic URL that should remain indexed.
2. Add an optional `legacyPath` or `routeAlias` concept to guides only if the guide engine needs to render at `/cities/[citySlug]/[seoPage]`.
3. Convert `/cities/[citySlug]/[seoPage]` to resolve a topic route to an admin guide first, then fall back to current programmatic rendering temporarily.
4. Keep `best-cafes`, `best-restaurants`, `best-beaches`, and `family-attractions` noindexed unless they have dedicated editorial guide content or future entity support.
5. When restaurants, hotels, and car rentals become real entities, use guide-engine listing blocks instead of creating another uncontrolled page generator.

## Risks Identified

- The current city topic route can create many pages from one template.
- Several topic pages are not directly admin-controlled and rely on keyword matching.
- Restaurant/cafe pages are premature until there is a restaurant entity or editorial guide source.
- Beach and family pages can duplicate destination/category intent.
- Future cleanup must not break indexed URL paths.
